# INA231 Basics and Register Map

This document summarizes how the INA231 works, how to scale its readings, and what each register address contains. It is intended for the 2S2P battery pack used to power the Raspberry Pi.

## 1) What the INA231 does
The INA231 is a current and power monitor that communicates over I2C. It measures:
- **Shunt voltage** across a resistor
- **Bus voltage** on the supply line
- **Current** (computed from shunt voltage and calibration)
- **Power** (computed from current and bus voltage)

## 2) Key measurement scaling (LSB values)
LSB (Least Significant Bit) is the value per bit. Raw register values are converted by multiplying by the LSB.

- **Shunt voltage register (0x01):** 2.5 µV/bit
- **Bus voltage register (0x02):** 1.25 mV/bit
- **Current register (0x04):** Current_LSB (A/bit, amps per bit), set by calibration
- **Power register (0x03):** 25 × Current_LSB (W/bit)

### Calibration
The calibration register sets the scale for current and power. After calibration:

- Current (A) = raw_current * Current_LSB
- Power (W) = raw_power * 25 * Current_LSB

Calibration register computation:

Cal = 0.00512 / (Current_LSB × R_shunt)

Where:
- R_shunt is the shunt resistor in ohms
- Current_LSB is the chosen current resolution in A/bit

## 3) Register map
| Address | Name | Description | Notes |
|---|---|---|---|
| 0x00 | Configuration | Conversion times, averaging, operating mode | Write during init |
| 0x01 | Shunt Voltage | Signed shunt voltage measurement | 2.5 µV/bit |
| 0x02 | Bus Voltage | Unsigned bus voltage measurement | 1.25 mV/bit |
| 0x03 | Power | Power measurement | 25 × Current_LSB |
| 0x04 | Current | Signed current measurement | Current_LSB |
| 0x05 | Calibration | Scaling for current/power | See formula |
| 0x06 | Mask/Enable | Alert and conversion ready flags | Optional |
| 0x07 | Alert Limit | Alert threshold register | Optional |
| 0xFE | Manufacturer ID | Device identification | Read-only |
| 0xFF | Die ID | Device identification | Read-only |

## 4) Alignment with main.c
This document aligns with the example in [docs/hardware/current&power-monitor-INA231/main.c](docs/hardware/current&power-monitor-INA231/main.c):

1. I2C address is 0x40, used as 0x40 << 1 in the HAL call.
2. Configuration register (0x00) is written with 0x4127.
3. Calibration register (0x05) is written with 168.
4. Raw values are read and scaled using:
	- Shunt voltage: 2.5 µV per bit
	- Bus voltage: 1.25 mV per bit
	- Current: Current_LSB = 0.000305 A/bit
	- Power: 25 × Current_LSB
5. UART output prints bus voltage, shunt voltage, current, and power every 2 seconds.
