# INA231 Basics and Register Map

This document summarizes how the INA231 works, how to scale its readings, and what each register address contains. It is intended for the 2S2P battery pack used to power the Raspberry Pi.

## 1) What the INA231 does
The INA231 is a current and power monitor with an I2C (Inter-Integrated Circuit) interface. It measures:
- **Shunt voltage** across an external shunt resistor (signed)
- **Bus voltage** on the supply side (unsigned)
- **Current** derived from shunt voltage and calibration (signed)
- **Power** derived internally from current and bus voltage (unsigned)

## 2) Key measurement scaling (LSB values)
LSB (Least Significant Bit) is the per-bit scale factor used to convert a raw register count into real units.

- **Shunt voltage register (0x01):** 2.5 µV/bit
- **Bus voltage register (0x02):** 1.25 mV/bit
- **Current register (0x04):** Current_LSB (A/bit), set by calibration
- **Power register (0x03):** 25 × Current_LSB (W/bit)

### Calibration formula
The calibration register sets the scaling for current and power. Use:

$$
\text{Cal} = \frac{0.00512}{\text{Current\_LSB} \cdot R_{shunt}}
$$

Where:
- $R_{shunt}$ is the shunt resistor value in ohms
- Current_LSB is the chosen current resolution in A/bit

**Practical guidance:**
1. Choose a maximum expected current for the 2S2P pack.
2. Select Current_LSB = Max_Current / 32768 (or a convenient round value).
3. Compute Cal and write it to register 0x05.

## 3) Register map
| Address | Name | Description | Notes |
|---|---|---|---|
| 0x00 | Configuration | Conversion times, averaging, operating mode | Write during init |
| 0x01 | Shunt Voltage | Signed shunt voltage measurement | 2.5 µV/bit |
| 0x02 | Bus Voltage | Unsigned bus voltage measurement | 1.25 mV/bit |
| 0x03 | Power | Power measurement | 25 × Current_LSB |
| 0x04 | Current | Signed current measurement | Current_LSB |
| 0x05 | Calibration | Scaling for current/power | See formula |
| 0x06 | Mask/Enable | Alert and conversion ready flags | Optional |
| 0x07 | Alert Limit | Alert threshold register | Optional |
| 0xFE | Manufacturer ID | Device identification | Read-only |
| 0xFF | Die ID | Device identification | Read-only |

## 4) Sign and range notes
- **Shunt voltage and current are signed** (two’s complement). Reverse current reads as negative.
- **Bus voltage and power are unsigned**.

## 5) Suggested integration steps (summary)
1. Configure I2C address and bus.
2. Write Configuration register (0x00).
3. Compute and write Calibration (0x05).
4. Read and scale shunt, bus, current, and power registers.
5. Validate readings against a multimeter or known load.
