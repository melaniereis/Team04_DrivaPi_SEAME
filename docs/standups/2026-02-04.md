## Challenges

**Problem:** GitHub Actions Security Vulnerability

* **Who:** Bernardo
* **Impact:** High (Potential for unauthorized repo modifications during checks)
* **Solution:** Although final merge approval is secure, the workflow allows content modification. Facilitators suggested implementing **signed attestations**.

**Problem:** Qt Metalayer Stale Builds

* **Who:** Melanie (with Gaspar)
* **Impact:** High (Cannot deploy new GUI changes)
* **Solution:** Debugging the bitbake/recipe configuration to ensure the build system picks up the new source files.

**Problem:** Burnt Servo Motor

* **Who:** Miguel
* **Impact:** Medium (Downtime for hardware repair)
* **Solution:** Disassembled vehicle and replaced the component immediately.

---

## Decisions

**GitHub Actions Security:**

* **Observation:** Verified that a user without ownership rights can modify repository content if the workflow has `contents: write` permissions.
* **Decision:** While our mandatory PR approval process prevents these changes from reaching `master`, we will investigate **signed attestations** to secure the artifacts and workflow further.
