# DrivaPi Brand Guidelines

This document outlines the standard usage for the DrivaPi logo to ensure visual consistency across all project documentation, code repositories, and presentations.

## 1. Logo Variants

**Primary Logo (Logotype Only)**
* **File:** `drivapi_logo_v3_white.png` 
* **Description:** Crimson "DrivaPi" text with the integrated berry over a white abstract car outline.
* **Usage:** This is the core identifier. Use this in the vast majority of cases: UI headers, document footers, code repository READMEs, and anywhere vertical space is limited. 

**Secondary Logo (Black Car Lockup)**
* **File:** `drivapi_logo_v3_black.png`
* **Description:** The text accompanied by a high-contrast black abstract car outline.
* **Usage:** Use this when establishing the mobility/automotive context of the project is necessary, such as the cover slide of a presentation or the main hero image of a project repository. It must only be placed on white or very light gray backgrounds.

**Tertiary Logo (Green Car Lockup)**
* **File:** `drivapi_logo_v3.jpg`
* **Description:** The text accompanied by a mint-green abstract car outline.
* **Usage:** Use sparingly. The red/green color combination is visually demanding. Limit this to specific team-branded assets where the primary brand colors need to be highlighted.

## 2. Clear Space
To ensure impact and readability, the logo must always be surrounded by a minimum clear space. No other text, UI elements, or graphics should enter this area.
* **Rule:** The clear space on all sides must be equal to the height of the capital "D" in the logo.

## 3. Minimum Size
To protect the legibility of the integrated Raspberry Pi icon, the logo must not be scaled below these thresholds:
* **Primary Logo (Text):** Minimum width of 120px.
* **Secondary/Tertiary Logos (Car):** Minimum width of 250px to preserve the thin lines of the car outline.

## 4. Unacceptable Usage
To maintain the integrity of the brand, strictly avoid the following:
* **Do not** stretch, squash, or alter the aspect ratio.
* **Do not** attempt to separate the car outline from the text to use it as a standalone icon; the line weights are not designed to stand entirely alone.
* **Do not** place the Secondary or Tertiary lockups over busy photographs or dark backgrounds where the car outline loses contrast. 
* **Do not** change the hex colors of the crimson text or the car outlines.

## 5. Background Contrast
* If placing the primary logo on a dark background where the black outlines disappear, you must use a truly white, solid-color version of the text (no outlines). The reverse must be done if the background is light.

## 6. SEA:ME Logo Usage
* The same logic is applied:
    - The white version is meant to be used on dark backgrounds.
    - The black version is meant to be used on light backgrounds.
